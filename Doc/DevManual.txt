//-----------------------------------------------------------------------------
// (1) Стилистические правила

(1.1) Имена директорий - с большой буквы, CamelCase.
(1.2) Названия проектов завершать суффиксом, обозначающим IDE или систему сборки.
Список суффиксов (пополняемый):
cb -- Code::Blocks;
msvc -- Microsoft Visual Studio C++.

Например: RnD_cb20x03.
Комментарий: чтобы файлы проектов не конфликтовали и можно было легко их удалить
через удаление папки.

//-----------------------------------------------------------------------------
// (2) Структура директорий
(2.1) Корневая директория проекта Zavod3D называется "Zavod3D" и содержит:
директории Doc, Sources; файлы LICENSE, README, и т.д.
Вы должны создать директорию с любым именем, в которую нужно склонировать репозиторий.
В терминлогии проекта Zavod3D эта директория называется "надкорневой". В надкорневой директории
необходимо создать поддиректории с именами Zavod3DExternal, Zavod3DBuild.
Таким образом, должна получиться структура:
<Директория, созданная вами (надкорневая директория)>
-- Zavod3D (Doc, Sources; файлы LICENSE, README, и т.д.)
-- Zavod3DBuild
-- Zavod3DExternal

Директория Zavod3DBuild служит для размещения выходных файлов, получающихся при сборке движка.
Директория Zavod3DExternal служит для размещения сторонних библиотек.
(2.2) Файлы проектов размещать в Zavod3D\Sources\ProjectFiles.

(2.3) Для объектных файлов в директории Zavod3DBuild\Intermediate создать поддиректорию с именем
проекта, а в ней - директорию с названием типа целевого проекта (Debug, Release и т.д.)
Путь из надкорневой директории будет выглядеть так например:
<надкорневая директория>\Zavod3DBuild\Intermediate\SomeProject_cb20x03\Debug

(2.4) Для целей сборки (исполняемых (exe) и dll файлов) назначить имя в директории Zavod3DBuild\Bin,
добавив суффикс с названием типа целевого проекта (Debug, Release и т.д.)
Например:
<надкорневая директория>\Zavod3DBuild\Bin\SomeProject_cb20x03_Debug.exe
